# Orb — Python FastAPI Backend

Handles **Pinecone RAG retrieval** and **Ollama embeddings**.  
AI chat responses are generated by Puter.js on the client — no LLM runs here.

---

## Architecture

```
User types message
       │
       ▼  Orb Widget (browser)
       │
       ├─ [Pinecone ON] ──► POST /chat
       │                      │
       │              embed via Ollama (nomic-embed-text)
       │                      │
       │              query Pinecone (cosine ≥ 0.55)
       │                      │
       │              return { context, sources }
       │
       ▼  buildPrompt(message + context)
       │
       ▼  window.puter.ai.chat(prompt)   ← Puter.js, free, no API key
       │
       ▼  Response shown in chat
```

## Running with Docker (recommended)

### 1. Copy and fill the env file

```bash
cp .env.example .env
# Edit .env — set PINECONE_API_KEY, PINECONE_INDEX_NAME, ALLOWED_ORIGINS
```

### 2. Start everything

```bash
# From the project root (where docker-compose.yml lives)
docker compose up -d
```

### 3. Pull the embedding model (once)

```bash
docker compose exec ollama ollama pull nomic-embed-text
```

### 4. Ingest your documents

```bash
# Put docs in backend/train-docs/ then:
curl -X POST http://localhost:8000/ingest-directory \
  -H "Content-Type: application/json" \
  -d '{"namespace": "default"}'
```

API docs: **http://localhost:8000/docs**

## Running bare-metal (no Docker)

```bash
cd backend
pip install -r requirements.txt
cp .env.example .env   # edit: set OLLAMA_URL=http://localhost:11434
# Make sure Ollama is running separately
ollama pull nomic-embed-text
uvicorn main:app --reload --port 8000
```

## Why Ollama for embeddings?

| OpenAI embeddings   | Ollama (nomic-embed-text) |
|---------------------|---------------------------|
| Costs per call      | Free — runs on your server |
| Requires API key    | No key needed              |
| Data leaves server  | Fully local / private      |
| 1536 dimensions     | 768 dimensions (still excellent for RAG) |

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `PINECONE_API_KEY` | ✓ | — | Your Pinecone API key |
| `PINECONE_INDEX_NAME` | ✓ | — | Index name (must match widget config) |
| `PINECONE_ENVIRONMENT` | | `us-east-1` | Pinecone region |
| `OLLAMA_URL` | | `http://localhost:11434` | Ollama server URL |
| `OLLAMA_EMBED_MODEL` | | `nomic-embed-text` | Embedding model name |
| `OLLAMA_EMBED_DIM` | | `768` | Vector dimension |
| `ALLOWED_ORIGINS` | ✓ | `http://localhost:3000` | Comma-separated allowed origins |
| `TRAIN_DOX_PATH` | | `./train-docs` | Default docs folder |

---

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/` | Health check — returns config summary |
| `POST` | `/chat` | Embed query → Pinecone → return context |
| `POST` | `/embed` | Return raw Ollama embedding |
| `POST` | `/ingest-directory` | Chunk, embed, upsert docs to Pinecone |
| `DELETE` | `/namespace/{ns}` | Delete all vectors in a namespace |

## CORS

Only domains in `ALLOWED_ORIGINS` can call the API:

```env
# Single domain
ALLOWED_ORIGINS=https://mysite.com

# Multiple (comma-separated)
ALLOWED_ORIGINS=https://mysite.com,https://app.mysite.com,http://localhost:3000
```

## Connecting the Next.js frontend

In your Next.js `.env.local`:

```env
BACKEND_URL=http://localhost:8000
```

`/api/chat` and `/api/embed` will automatically proxy to this backend.
